# Проект OOP Design

Сейчас класс в Zoo не имеет никаких методов, и из за этого тесты не проходят, нам нужно чтобы они проходили и для этого нужно дописать класс Zoo

## Структура проекта

```
oop-design/
├── src/                    # Основной исходный код
│   ├── __init__.py
│   ├── animal.py          # Абстрактный базовый класс для всех животных
│   ├── animals.py         # Конкретные реализации животных
│   └── zoo.py             # Класс управления зоопарком
├── tests/                 # Модульные тесты
│   └── test_animals.py    # Набор тестов
├── docs/                  # Документация
├── main.py               # Основное приложение
├── requirements.txt      # Зависимости проекта
├── .gitignore           # Файл исключений Git
└── README.md            # Этот файл
```

## Демонстрируемые принципы OOP

### 1. **Инкапсуляция**
- Приватные атрибуты (с префиксом `_`)
- Публичные свойства и методы
- Контролируемый доступ через getter'ы

### 2. **Наследование**
- `Animal` - абстрактный базовый класс
- Конкретные реализации: `Dog`, `Cat`, `Bird`
- `Zoo` класс с композицией

### 3. **Полиморфизм**
- Абстрактные методы `make_sound()` и `eat()`
- Разные реализации в каждом классе животного
- Переопределение методов

### 4. **Абстракция**
- Использование `ABC` (Abstract Base Class)
- Абстрактные методы, которые должны быть реализованы подклассами
- Скрытие деталей реализации

### 5. **Композиция**
- `Zoo` класс управляет коллекцией объектов `Animal`
- Отношение "имеет" (Zoo имеет Animals)

### 6. **Принцип единственной ответственности**
- Каждый класс имеет одну четкую цель
- `Animal`: Базовое поведение животного
- `Dog/Cat/Bird`: Поведение, специфичное для вида
- `Zoo`: Управление коллекцией

## Возможности

### Классы животных
- **Dog**: Обучается трюкам и может их выполнять
- **Cat**: Управляет настроением, может царапать предметы
- **Bird**: Может летать на разные высоты (кроме пингвина!)

### Управление зоопарком
- Добавление/удаление животных
- Кормление всех животных одновременно
- Физическая нагрузка для всех животных
- Поиск животных по имени
- Получение количества животных по виду
- Отображение информации обо всех животных

## Характеристики качества кода

### Подсказки типов
Все функции и методы включают подсказки типов для лучшей ясности кода и поддержки IDE.

```python
def eat(self, food: str) -> None:
    """Накормить животное."""
```

### Docstrings
Полные docstrings, следующие руководству Google:
- Документация класса
- Документация методов
- Документация параметров
- Документация возвращаемых значений

### Декораторы свойств
Безопасный доступ к приватным атрибутам:
```python
@property
def name(self) -> str:
    """Получить имя животного."""
    return self._name
```

### Модульные тесты
Полное покрытие тестами, включая:
- Инициализацию объектов
- Поведение методов
- Граничные случаи
- Управление коллекциями

## Пример использования

```python
from src.animals import Dog, Cat, Bird
from src.zoo import Zoo

# Создать зоопарк
zoo = Zoo("Мой парк", "Нью-Йорк")

# Создать животных
dog = Dog("Rex", 5, "Golden Retriever")
cat = Cat("Whiskers", 3, "Orange Tabby")

# Добавить в зоопарк
zoo.add_animal(dog)
zoo.add_animal(cat)

# Использовать животных
dog.teach_trick("Sit")
print(dog.perform_trick("Sit"))

# Управлять зоопарком
zoo.feed_all()
zoo.display_animals()
```
